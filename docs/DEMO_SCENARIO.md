# Scenariusz demo

## Cel demo

Pokazac, ze zespol potrafi przewidziec wysokie godziny generacji OZE i przelozyc je na praktyczna informacje dla operatora sieci.

## Narracja 2-minutowa

1. Wybieramy kilka lokalizacji z obszaru Tauron.
2. System pobiera prognoze pogody i estymuje produkcje PV/wiatr.
3. Dla kazdej lokalizacji liczymy risk score.
4. Dashboard pokazuje, gdzie jutro w godzinach poludniowych ryzyko jest najwyzsze.
5. Operator widzi rekomendacje: magazynowanie, przesuniecie zuzycia, monitoring albo inwestycja.

## Przykladowa historia

Jutro prognozowane jest wysokie naslonecznienie i niskie zachmurzenie. Model wskazuje, ze w godzinach 11:00-14:00 kilka lokalizacji bedzie miec wysoka produkcje PV. Tam, gdzie publiczne proxy pokazuje niski margines przylaczeniowy, system oznacza wysoki poziom ryzyka.

## Co klikamy w demo

- Start: mapa z lokalizacjami.
- Klikniecie lokalizacji: wykres produkcji na najblizsze godziny.
- Przelaczenie godziny: zmiana kolorow ryzyka na mapie.
- Panel rekomendacji: konkretne dzialanie dla operatora.
- Slajd koncowy: jak dane Taurona poprawiaja dokladnosc.

## Najwazniejsze zdanie do jury

Z danych publicznych pokazujemy dzialajacy demonstrator predykcyjny. Z danymi Taurona ten sam mechanizm moze wspierac rzeczywiste decyzje dotyczace przylaczen, inwestycji i elastycznosci sieci.
